# 🧠 Algorithmic Patterns

MAANG interviews heavily test your ability to recognize **patterns** rather than just knowing data structures. If you can identify the underlying pattern of a problem, the solution becomes much easier to formulate.

This directory contains study guides for the most common algorithmic patterns.

## Available Patterns

1. **[Sliding Window](sliding_window.md)**
   - *Used for*: Finding a subset of elements (subarray, substring) that satisfies a certain condition.
2. **Two Pointers** (Coming soon)
   - *Used for*: Searching pairs in a sorted array, or processing two strings/arrays simultaneously.
3. **Fast & Slow Pointers** (Coming soon)
   - *Used for*: Cycle detection in Linked Lists or Arrays.
4. **Merge Intervals** (Coming soon)
   - *Used for*: Overlapping intervals, scheduling.
5. **Cyclic Sort** (Coming soon)
   - *Used for*: Finding missing numbers in an array containing numbers in a given range.
6. **In-place Reversal of a LinkedList** (Coming soon)
   - *Used for*: Reversing nodes without using extra space.
7. **Tree BFS & DFS** (Coming soon)
   - *Used for*: Level-order traversal (BFS) or depth-first searching (DFS).
8. **Two Heaps** (Coming soon)
   - *Used for*: Finding the median of a number stream or similar dynamic tracking.
9. **Topological Sort** (Coming soon)
   - *Used for*: Finding a valid ordering of dependent tasks (Graphs).

## How to use this directory

1. Read the markdown explanation for a pattern.
2. Study the provided Jupyter notebook (if available in `/notebooks`).
3. Solve the linked problems in the `topics/` directory that utilize this pattern.
