# System Design Preparation

System Design interviews test your ability to architect scalable, reliable, and maintainable software systems. While DSA tests your micro-level coding skills, System Design tests your macro-level engineering judgment.

## How to use this guide
1. **Understand the Core Concepts**: Before attempting to design Twitter or Netflix, you must understand the building blocks (Load Balancers, Caching, Databases, etc.). Read the `01_core_concepts` folder first.
2. **Study the Common Problems**: The `02_common_problems` folder contains the classic MAANG interview questions. Study the step-by-step framework used to solve them.
3. **Practice the Framework**: 
   - **Step 1:** Clarify Requirements (Functional vs. Non-Functional)
   - **Step 2:** Back-of-the-envelope Estimation (QPS, Storage, Bandwidth)
   - **Step 3:** High-Level Design (Draw the architecture)
   - **Step 4:** Deep Dive (Identify bottlenecks and optimize)
